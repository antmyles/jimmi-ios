# Wearable Integration Alternatives Research Notes

## Official platform APIs

### Android Health Connect
Source: https://developer.android.com/health-and-fitness/health-connect

Android Health Connect is positioned as a platform that simplifies connectivity between Android health and fitness apps. It lets apps, with user permission, combine health and fitness data to generate insights. The page notes that Google Fit APIs are transitioning away starting in 2026, making Health Connect the forward-looking Android route. It supports standard data-management operations, health and fitness data types, background reading, and Android SDK 28+ compatibility.

### Apple HealthKit
Source: https://developer.apple.com/documentation/healthkit

Apple HealthKit provides a central repository for health and fitness data on iPhone and Apple Watch. With user permission, apps communicate with the HealthKit store to access and share this data. It is appropriate when JIMMI has an iOS app and wants direct Apple Health / Apple Watch data without paying an aggregator.

## Managed aggregator pricing found so far

### Terra
Source: https://tryterra.co/pricing

Terra Quick Start is listed as starting at $499/month monthly or $399/month when billed annually, with 100,000 credits/month and a 30-day money-back guarantee.

### Spike API
Source: https://www.spikeapi.com/pricing

Spike Wearables & IoT API Sandbox starts from $450/month and includes unlimited API/SDK integrations, access to 40+ providers and 500+ devices, standardized data, historical retrieval, deduplication, and compliance positioning.

### ROOK
Source: https://www.tryrook.io/pricing

ROOK Core is listed at $399/month for up to 750 active users, with all integrations in ROOKConnect, multiple data sources per user, developer/admin portal, sandbox, and email/intercom support. ROOK also advertises an early-stage startup program.

### WHOOP direct API
Source: https://developer.whoop.com/api-terms-of-use/

WHOOP’s API Terms of Use state that access to and use of the APIs is currently provided at no charge, with prior notice if WHOOP decides to charge in the future. This makes WHOOP a potentially strong direct integration for beta users who already have WHOOP devices, though it only covers WHOOP rather than the full wearable market.

### Oura direct API
Source: https://api.ouraring.com/docs/

The Oura API provides access to data generated by the Oura Ring. To use it, JIMMI would need an Oura account and an API application. For multi-user use, Oura requires registering an OAuth2 application. By default, API applications have a ten-user limit and need approval before wider release.

### Withings Public API
Source: https://developer.withings.com/developer-guide/v3/withings-solutions/app-to-app-solution/

Withings states that its Public API allows individuals and companies, with or without a Withings contract, to access data from Withings users who consent to share their information. The flow uses OAuth, device data is transmitted to Withings servers, and partners retrieve health data using access tokens. Withings also describes a notification/webhook system for real-time updates.

### Garmin Health API
Source: https://developer.garmin.com/health-api/overview/

Garmin Health API provides all-day metrics such as steps, heart rate, sleep, stress, pulse ox, body battery, respiration, body composition, and blood pressure in JSON format. Garmin describes use cases such as Corporate Wellness, Population Health, and Patient Monitoring. The page states that after approval, developers get access to an evaluation environment, and that commercial use requires a license fee payment.
